<threat_model>
## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| dictionary copy → JSON-LD serialization | Author-controlled strings flow into `<JsonLd>` which sets innerHTML |
| dictionary copy → SSR HTML | Author-controlled strings flow into rendered page markup |

## STRIDE Threat Register (ASVS L1)

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-02-01 | Tampering/XSS | JSON-LD emission via `<JsonLd>` for new faqPageGraph/servicesGraph | mitigate | `<JsonLd>` already escapes `<` → `&lt;` before `dangerouslySetInnerHTML` (Phase 1 D-10). New schema flows through the same component; do not bypass it or hand-build `<script>` tags. |
| T-02-02 | Information disclosure | New lead/FAQ copy | accept | Copy is curated public business info sourced from site.ts NAP; no secrets, no PII. |
| T-02-03 | Tampering | Author-controlled FAQ answer text rendered in `<dd>`/`<Accordion>` | accept | React JSX text interpolation escapes by default; no `dangerouslySetInnerHTML` on visible copy. |

No unmitigated HIGH. No package installs in this plan (no Package Legitimacy Gate needed).
</threat_model>
