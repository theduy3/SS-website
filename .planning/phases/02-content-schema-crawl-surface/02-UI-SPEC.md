## Spacing Scale

8-point scale. All spacing values must be multiples of 4px. These token names map to Tailwind utilities (`p-4` = 16px, etc.).

| Token | Value | Tailwind | Usage |
|-------|-------|----------|-------|
| xs | 4px | `gap-1`, `p-1` | Inline micro-gaps |
| sm | 8px | `gap-2`, `p-2` | Tight element spacing |
| md | 16px | `p-4`, `gap-4` | Default element spacing, `px-6` section padding |
| lg | 24px | `gap-6`, `p-6` | Section horizontal padding (`px-6`) |
| xl | 32px | `gap-8`, `p-8` | Content group separation |
| 2xl | 48px | `gap-12`, `p-12` | Intra-section gaps |
| 3xl | 64px | `py-20` (80px), `py-28` (112px) | Section vertical padding |

**Exceptions:**
- Section vertical padding follows the established site pattern: `py-20 md:py-28` (80px / 112px) — NOT strict 64px. Use this for all content sections in new pages.
- `PageHeader` band uses `py-20 md:py-28` with `max-w-7xl px-6` — replicate exactly on the Laval page header.
- Touch targets for links/buttons: minimum 44px height (satisfied by `py-3` on Button = 12px + line height).

## Typography

All font sizes use Tailwind responsive utility classes. The site uses **two fonts**: Space Grotesk (body) and Archivo Black (headings). Do not introduce additional font weights or families.

| Role | Size | Responsive | Font | Weight | Line Height | Tailwind |
|------|------|-----------|------|--------|-------------|---------|
| Display / H1 | 36px → 60px | `text-4xl md:text-6xl` | Archivo Black | 400 (only weight available) | 1.05 (set globally) | `text-4xl md:text-6xl` |
| Heading H2 | 24px → 36px | `text-2xl md:text-3xl` | Archivo Black | 400 | 1.05 | `text-2xl md:text-3xl` |
| Body | 16px (default) / 18px (lead) | `text-base` / `text-lg` | Space Grotesk | 600 (base body weight) | `leading-relaxed` (~1.625) | `text-lg leading-relaxed text-mocha` for lead/answer-first contexts; implicit `text-base` otherwise |
| Label / Meta | 14px | `text-sm` | Space Grotesk | 600 | default | `text-sm uppercase tracking-widest text-mocha` |

**Heading transforms:** All headings (`h1`–`h4`) are globally `text-transform: uppercase; letter-spacing: -0.02em; line-height: 1.05` via `@layer base`. Do not override these on any heading in this phase.

**Two effective weights in use:** 400 (Archivo Black — headings only) and 600 (Space Grotesk — everything else, set as `font-weight: 600` on `body`). Do not introduce 700/800 or any other weight.

## Element-Level Design Contracts

### 1. Answer-First Lead Paragraph (CONTENT-01, D-03/D-04)

**What it is:** A single `<p>` tag, 40–60 words, factual direct-answer tone, SSR-rendered before any other page content (immediately after the opening `<>` fragment, before any `<JsonLd>` or `<section>` wrappers).

**Placement:** Must appear in raw SSR HTML as the first readable text element on the page. Precedes `<PageHeader>` on pages that use `<PageHeader>`. Precedes the hero `<section>` on the home page.

**Tailwind classes:**
```
<p className="sr-only">
  {dict.page.lead}
</p>
```

Wait — correction per D-03: the lead is NOT a visually hidden element. It IS the first visible paragraph. Apply:

```
<p className="mx-auto max-w-3xl px-6 pt-8 text-lg leading-relaxed text-mocha md:pt-12">
  {dict.page.lead}
</p>
```

**Exact contract:**
- `max-w-3xl` — constrains to 48rem, matching body-content width
- `px-6` — standard section horizontal padding
- `pt-8 md:pt-12` — 32px / 48px top clearance from header (do not use `py-*` — the `PageHeader` below it provides bottom spacing)
- `text-lg leading-relaxed text-mocha` — body-lead size, 1.625 line-height, dark-gray (not black)
- No background, no border, no card, no callout box — plain paragraph on `beige` background
- Wrapped in `<Reveal>` for consistent entry animation

**Pages that receive this block (D-04):**
1. Home (`src/app/[lang]/page.tsx`) — insert before hero `<section>`
2. Services index (`src/app/[lang]/services/page.tsx`) — insert before `<PageHeader>`
3. Each `/services/[slug]` — insert before the hero `<section>`
4. FAQ hub (`src/app/[lang]/faq/page.tsx`) — insert before `<PageHeader>`
5. Laval page (new `src/app/[lang]/laval/page.tsx`) — insert before `<PageHeader>`
6. About (`src/app/[lang]/about/page.tsx`) — insert before `<PageHeader>`

**Dictionary key convention:** `dict.{page}.lead` — e.g. `dict.home.lead`, `dict.faq.lead`, `dict.laval.lead`, `dict.about.lead`, `dict.servicesPage.lead`, `dict.serviceDetails[id].lead`.

**RTL (Arabic):** The `dir="rtl"` attribute is set on `<html>` by `layout.tsx` via `dirFor(lang)`. No additional `dir` or `text-align` override is needed on the `<p>` element — Tailwind's `px-6` padding and text alignment inherit correctly under RTL. Verify visually that `max-w-3xl` still constrains and `px-6` reads as start/end padding (not left/right) under `dir="rtl"`. No code change needed; this is a verification checkpoint.

### 2. Expanded FAQ Hub (CONTENT-02, D-01/D-02)

**Route:** existing `src/app/[lang]/faq/page.tsx` — expand in place.

**Accordion visual contract (unchanged from existing implementation):**
```
<div className="divide-y divide-tan/30 border-y border-tan/30">
  <details className="group py-4">
    <summary className="flex cursor-pointer list-none items-center justify-between gap-4 text-lg font-semibold">
      {item.q}
      <span className="text-tan transition-transform group-open:rotate-45" aria-hidden>+</span>
    </summary>
    <p className="mt-3 leading-relaxed text-mocha">{item.a}</p>
  </details>
</div>
```

**Do not change any of these classes.** The divider is `divide-tan/30`, the question is `text-lg font-semibold`, the answer is `leading-relaxed text-mocha`, the expand icon is `text-tan` with `rotate-45` on open.

**Content expansion contract:**
- Grow `dict.faq.items` from current count to 8–15 items
- Topics: hours, booking, parking (CF Carrefour Laval), services offered, pricing, walk-ins
- Answer length: 1–3 sentences per item, factual, no marketing language
- All 4 locales (en/fr/es/ar) must have identical item count and equivalent content

**Schema verbatim match (SCHEMA-02):** `faqPageGraph(dict.faq.items)` already feeds the same array into schema and `<Accordion>`. This contract is satisfied by construction — keep single-source pattern, never duplicate copy.

**Section wrapper (existing, do not change):**
```
<section className="mx-auto max-w-3xl px-6 py-16 md:py-24">
  <Accordion items={dict.faq.items} />
</section>
```

**RTL:** `<details>`/`<summary>` layout uses `justify-between` — in RTL the `+` indicator appears on the left (logical start), question text on the right. This is correct RTL behavior. The `rotate-45` transition is direction-agnostic. No override needed.

**Answer-first lead paragraph:** Add `dict.faq.lead` before `<PageHeader>` per the answer-first contract above.


**Location facts block (address/parking/transit/landmarks):**
- Use a `<dl>` (definition list) pattern — matches existing `<dl>` usage in `[slug]/page.tsx` price/duration block
- Each fact: `<dt className="text-sm uppercase tracking-wide text-mocha">` + `<dd className="mt-1 leading-relaxed text-mocha">`
- Section uses `<Reveal>` wrapper, no `bg-fog` (white background to alternate with the fog FAQ section below)

**Laval FAQ block:**
- `bg-fog` section, matches the existing per-service FAQ pattern
- `<h2 className="text-2xl text-espresso md:text-3xl">` for the FAQ heading
- `<Accordion items={dict.laval.faq.items} />` — same component, same visual contract
- 3–5 Laval-specific Q/As: CF Carrefour location, parking, transit, walk-in availability, nearby landmarks

**CTA block:**
- Matches existing pattern from `[slug]/page.tsx` bottom CTA: centered, `text-lg text-mocha` prompt, then `flex gap-4` row with `<Button>` (solid) + phone `<a>` (no button styling)

**PageHeader reuse:**
- `<PageHeader title={dict.laval.heading} intro={dict.laval.intro} />` — pass both props
- `intro` becomes the sub-headline in the `bg-fog` header band (existing `<p className="mt-6 max-w-2xl leading-relaxed text-mocha">`)

**Primary focal point:** The `<PageHeader>` heading (`dict.laval.heading`) in Archivo Black on the `bg-fog` band, directly below the answer-first lead paragraph — this is the dominant visual anchor of the page.

**RTL:** No special overrides. The `←`/`→` back-link pattern from `[slug]/page.tsx` — if a back-link to `/services` or home is added, use `{dirFor(lang) === "rtl" ? "→" : "←"}` as already established. Do not hardcode `←`.
