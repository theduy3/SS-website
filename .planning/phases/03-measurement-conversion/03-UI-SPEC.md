## Surface Specifications

### Surface 1: Consent Mode v2 Accept/Decline Bar

**Purpose:** Law 25 / Consent Mode v2. Default-denied; Accept enables `analytics_storage`. Decline keeps storage denied.

**Placement:** Fixed bottom of viewport. `position: fixed; bottom: 0; left: 0; right: 0`.

**Z-index:** `z-40` — below Header (`z-50`) and popup layer (`z-[100]`). Consent bar must not occlude the mobile nav menu.

**Visibility:** Server-rendered as `hidden` by default (consent already recorded or JS not loaded); shown via JS after consent state check. Must NOT be in the SSR HTML as visible — it would cause CLS and confuse crawlers.

**RTL:** `dir` attribute inherited from `<html lang>` / `<html dir>`. Use logical CSS properties where possible: `ps-4 pe-4` (padding-inline-start/end) or explicit `px-4`. Button order in LTR: [Decline] [Accept]. In RTL: order reverses naturally via `flex-row-reverse` or `dir="rtl"` on the bar container.

**Layout (mobile-first):**
- Mobile: stacked vertically — copy above, buttons below, full-width bar
- Desktop (md+): single row — copy left, buttons right, `justify-between`
- Bar height: auto, minimum 44px total tap area on buttons
- Max width: `max-w-7xl mx-auto` inside the fixed bar (matches page layout)

**Anatomy:**
```
[bar: bg-espresso text-cream, fixed bottom, z-40, py-3 px-4]
  [copy: text-sm] "{dict.consent.body}"
  [buttons: flex gap-3 items-center]
    [Decline: Button variant="outline" — border-cream/40 text-cream hover:bg-cream/10]
    [Accept: custom — bg-cream text-espresso hover:bg-tan, rounded-pill px-6 py-2 text-sm font-semibold uppercase tracking-wide]
```

Note: `Button.tsx` renders an `<a>` or `<Link>` (href-based). Consent buttons need `<button type="button">` — use the same visual classes as `Button` but as a native `<button>`, not the `Button` component. Do NOT add an `onClick` prop to `Button.tsx`.

**Consent state storage:** SSR-safe cookie (not localStorage). Cookie name: `ss_consent`. Values: `"granted"` | `"denied"`. Read server-side for SSR safety. SameSite=Lax; no Secure flag needed (analytics-only, non-sensitive).

**Copy:** See Copywriting Contract below. All 4 locales via dictionary.

**Interaction states:**
- Default: bar visible (consent not yet recorded)
- After Accept: bar unmounts; `gtag('consent','update',{analytics_storage:'granted'})` fires
- After Decline: bar unmounts; consent stays denied (no gtag update)
- Subsequent page loads: bar hidden (cookie present)

### Surface 2: Sticky Mobile-Only Call + Book Bar

**Purpose:** Always-visible bottom CTA on key pages for mobile visitors. Conversion surface for `phone_click` and `book_cta_click` events.

**Placement:** Fixed bottom of viewport. `position: fixed; bottom: 0; left: 0; right: 0`.

**Visibility:** `md:hidden` — mobile-only, hidden on tablet/desktop breakpoint. Desktop visitors use the Header's Book button.

**Key pages only (D-09):** home, services index, `/services/[slug]`, FAQ hub, `/laval`. Not rendered on appointments, contact, legal pages.

**Always visible (D-09):** No hide-on-scroll. `position: fixed` always.

**Z-index:** `z-40` — same tier as consent bar. When both bars are present simultaneously (first visit, key page, mobile), they stack. Consent bar takes `z-40` and the CTA bar takes `z-[39]` OR the CTA bar is suppressed while consent bar is visible (preferred — avoid double-bar clutter). Implementation preference: hide CTA bar while consent cookie is absent; show after consent is recorded.

**Coexistence with Header (z-50):** CTA bar is BELOW Header (`z-40` < `z-50`). No conflict.
**Coexistence with popup layer (z-[100]):** CTA bar is BELOW popup (`z-40` < `z-[100]`). No conflict.

**RTL:** Two-button layout. In LTR: [Call] [Book]. In RTL: [Book] [Call] (logical reading order). Achieve via `flex-row-reverse` on RTL locales OR `flex` with button DOM order swapped per `dir`.

**Layout:**
```
[bar: bg-espresso text-cream, fixed bottom, z-40, py-3 px-4, md:hidden]
  [inner: flex gap-3 items-center justify-center max-w-7xl mx-auto]
    [Call button: Button href={site.contact.phoneHref} variant="light" className="flex-1 min-h-[44px]"]
      {dict.cta.callNow}
    [Book button: Button href={localizedHref(site.booking)} variant="light" className="flex-1 min-h-[44px]"]
      {dict.cta.book}
```

Note: Both buttons `variant="light"` on `bg-espresso` background. `cream` buttons on `espresso` bar is the established Header pattern (`dict.cta.book` Book button in mobile nav uses `variant="light"`).

**Body padding:** Pages that render the sticky bar must add `pb-[64px] md:pb-0` to the `<main>` or page wrapper to prevent the bar from covering content at the bottom of the page.

**Event wiring (MEAS-02):** `phone_click` fires on Call button click (via `onClick` wrapper on the `<a>` rendered by `Button`). `book_cta_click` fires on Book button click. Wiring lives in a wrapper component, not in `Button.tsx` itself.

### Surface 3: Above-Fold Trust Band

**Purpose:** Credibility signal above the fold on key pages. Rating + review count as primary; "Since 2024" as secondary (D-08).

**Placement:** Below the hero / page header section, above the first content section. Inline in the page SSR HTML — not fixed/sticky.

**SSR requirement:** Must be in server-rendered HTML. No client-only rendering. No `opacity:0` / Framer `<Reveal>` wrapper. Crawler must see it.

**Live-data gate (SCHEMA-03 pattern):** Render rating + reviewCount ONLY when `googleReviews.aggregate.ratingValue` is present (i.e., live data fetched). If not present, show "Since 2024" only. Do NOT show a placeholder rating. Mirror the `reviewsFetchedAt` guard from `src/lib/seo.ts`.

**"Since 2024" source:** `site.established` (add `established: 2024` to `src/lib/site.ts`). Render as static string — no `new Date()`, no arithmetic. The rendered string is the dictionary key `dict.trust.established` → "Since 2024" (en), "Depuis 2024" (fr), "Desde 2024" (es), "منذ 2024" (ar).

**Layout:**
```
[band: bg-fog py-2 px-6]
  [inner: flex flex-wrap items-center gap-3 max-w-7xl mx-auto]
    [IF live data]
      [Stars className="text-espresso h-4 w-4"] (scale down from default h-6 w-6 via className override)
      [rating: text-lg font-semibold text-espresso] "{ratingValue}"
      [separator: aria-hidden text-tan] "·"
      [reviewCount: text-sm text-mocha] "{reviewCount} {dict.trust.reviewsWord}"
      [separator: aria-hidden text-tan] "·"
    [/IF]
    [established: text-sm text-tan] "{dict.trust.established}"
```

**Accessibility:** Stars are `aria-hidden` (per existing `Stars.tsx`). Rating number is visible text — sufficient. `<span aria-hidden>·</span>` separators. No additional ARIA needed.

**RTL:** `flex-wrap` + `gap-3` layout reads correctly in RTL. `dir` inherited from page. No special RTL treatment needed beyond dictionary strings.

**Width:** Full bleed band with `max-w-7xl mx-auto` inner container — matches Header, PageHeader, Footer layout.

### Empty states

**Trust band (no live data):** Show `dict.trust.established` only ("Since 2024"). No empty-state heading — the band simply shows less content. No "loading" state — SSR only.

**Consent bar:** No empty state. Bar is either present (consent not recorded) or absent (consent recorded).

**Sticky CTA bar:** Always present on key pages when rendered. No empty state.

### Error states

**Consent storage failure:** If the `ss_consent` cookie cannot be set (e.g. storage blocked), default behavior is safe: consent remains denied (Consent Mode v2 default-denied). No error UI shown to the user — fail silently toward privacy protection.

**GA4 tag load failure:** `gtag` calls are no-ops if the script fails to load (guard with `typeof gtag !== 'undefined'` check before firing events). No error UI.

**ContactForm error state:** Already handled by `ContactForm.tsx` (`text-red-700` inline error). No change this phase.

### Destructive actions

None in this phase. Declining consent is not destructive — it is the privacy-protective default. No confirmation dialog needed for Decline.

## Interaction Contract

### Consent Bar Interactions

| Trigger | Action |
|---------|--------|
| Page load, `ss_consent` cookie absent | Bar renders (JS-controlled, not SSR) |
| User clicks Accept | Set `ss_consent=granted` cookie; call `gtag('consent','update',{analytics_storage:'granted'})`; unmount bar |
| User clicks Decline | Set `ss_consent=denied` cookie; no gtag call; unmount bar |
| Page load, `ss_consent=granted` | Bar hidden; GA4 fires normally |
| Page load, `ss_consent=denied` | Bar hidden; GA4 no-ops |
| `prefers-reduced-motion` | No motion animations on bar mount/unmount — use `display` toggle, not CSS transitions |

### Sticky CTA Bar Interactions

| Trigger | Action |
|---------|--------|
| Call button click | `phone_click` GA4 event fires; browser initiates `tel:` call |
| Book button click | `book_cta_click` GA4 event fires; navigate to `/{locale}/appointments` |
| Consent cookie absent | Bar hidden (avoid double-bar); shows after consent recorded |
| Viewport ≥ md (768px) | Bar hidden via `md:hidden` |

### Trust Band Interactions

Static — no interactions. No hover states, no click behavior. Pure SSR display element.

## Animation & Motion Contract

**Consent bar:** No entrance/exit animation. Mount/unmount via conditional render or `display: none` toggle. Rationale: CLS discipline — animated bars shift page layout and fail INP budget.

**Sticky CTA bar:** No animation. Fixed position, always present when rendered. No slide-in on scroll.

**Trust band:** No animation. Static SSR HTML. Must NOT be wrapped in `<Reveal>` or any Framer Motion component — would hide content from AI crawlers (Phase 2 pitfall, carried forward).

**General rule:** `prefers-reduced-motion: reduce` must result in zero motion for all three surfaces.

---

## CLS / INP / CWV Discipline

These constraints are load-bearing for MEAS-04 success criteria.

| Rule | Rationale |
|------|-----------|
| Trust band is SSR HTML, no layout shift | Avoids CLS from late-loading content injection |
| Consent bar is JS-mounted (not SSR) | Avoids CLS from unexpected bar on first paint; bar appears after hydration |
| Consent bar and CTA bar use `position: fixed` | Fixed elements do not affect document flow; zero layout shift |
| No `new Date()` in trust band | Deterministic render; no hydration mismatch |
| Stars component size override (`h-4 w-4`) via className | Prevents layout shift from default `h-6 w-6` in trust band context |
| CTA bar body padding `pb-[64px] md:pb-0` | Prevents bar from covering last-page-element; prevents scroll-to-bottom CLS |
| GA4 `next/script strategy="afterInteractive"` | Does not block FCP/LCP |
| `web-vitals` reporter is client-only, consent-gated | Does not affect SSR HTML or LCP |
