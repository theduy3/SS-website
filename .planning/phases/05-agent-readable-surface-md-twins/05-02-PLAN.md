## STANDALONE_PATHS reconciliation (Rule 7 — surfaced, not papered over)

CONTEXT D-02 and milestone success-criterion 2 say `.md` routes must be "registered in `STANDALONE_PATHS`." RESEARCH found the proxy `matcher` regex already excludes ALL dotted paths (`.*\..*` lookahead), so `.md` routes bypass the proxy function entirely — `STANDALONE_PATHS` entries for them would be 96 lines of dead config with zero functional effect. CONTEXT's own Claude's-Discretion block calls registration "the belt-and-suspenders merge-gate invariant, not a functional requirement."

**Decision (documented here):** The load-bearing merge gate is the **`proxy.test.ts` passthrough assertion** (Task 2) — it is what EXP-03 literally requires ("with a `proxy.test.ts` passthrough assertion") and what fails loudly if the matcher is ever changed to start processing `.md` paths. We **skip the 96 `STANDALONE_PATHS` entries** per the RESEARCH rationale (dead config, maintenance cost, no benefit). `/llms.txt`'s existing single `STANDALONE_PATHS` entry is left untouched (removing it would break its existing test). The EXP-03 invariant is fully satisfied by the passing `proxy.test.ts` assertion + the `md-coverage.test.ts` parity gate.
