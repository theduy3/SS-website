<objective>
Build the dependency-free foundation for the `.md` twin surface and de-risk the one MEDIUM-confidence assumption (the `app/[lang].md/` home-folder convention) by smoke-testing a single handler before fanning out to all 14.

This plan delivers: (1) a single shared date source (`page-dates.ts`) so the `.md` `updated` frontmatter (D-05) never drifts from `sitemap.ts`; (2) `md-routes.ts` `allMdPaths()` — the canonical route enumerator that feeds the coverage-parity gate (D-02); (3) the full `md-serializer.ts` with shared helpers and one `renderXxxMd()` per route family — a pure module unit-tested in isolation (no `server-only`); (4) the home `[lang].md/route.ts` handler, smoke-tested so `/en.md` returns HTTP 200 clean text BEFORE Plan 02 creates the other 13 handlers.

Purpose: The serializer + route-source are imported by every Plan-02 handler; building and testing them in isolation here keeps Plan 02 a mechanical fan-out. Smoke-testing the home folder first means a folder-convention failure (RESEARCH assumption A1, MEDIUM confidence) surfaces against ONE file, not 14. Satisfies EXP-03 ("force-static, content derives from the same dictionary source as the HTML page — no drift").
Output: `src/lib/page-dates.ts`, `src/lib/md-routes.ts`, `src/lib/md-serializer.ts` (+ test), `src/app/[lang].md/route.ts`, and `sitemap.ts` refactored to import the extracted dates.
